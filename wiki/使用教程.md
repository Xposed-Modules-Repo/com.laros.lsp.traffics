# TrafficManager Wiki: 使用教程

## 1. 准备环境

- Android 10+
- 已安装 LSPosed（建议 Zygisk）
- 建议双卡设备

## 2. 安装

1. 安装 `app-debug.apk`
2. 在 LSPosed 启用模块
3. 勾选推荐作用域：
   - `com.android.phone`
   - `com.xiaomi.phone`
   - `com.qti.phone`
   - `com.qualcomm.qti.telephonyservice`
   - `com.android.services.telephony`
   - `com.android.telephony`
4. 重启设备

## 3. 首次配置

1. 打开 TrafficManager
2. 授予定位、附近设备（Wi-Fi）、通知等权限
3. 点击“刷新状态”确认可读到 SSID/BSSID
4. 使用“学习当前 Wi-Fi -> SIM1/SIM2”快速写规则
5. 点击“启动服务”

## 4. 关键配置项说明

- `priority`: 数值越大优先级越高
- `cooldownSec`: 防抖，避免短时间重复切卡
- `leaveDelaySec`: 离开 Wi-Fi 后延迟回切
- `leaveMissThreshold`: 连续多少次未匹配才判定离开
- `revertOnLeave`: 离开 Wi-Fi 时是否回到进场前卡槽

## 5. 验证流程

1. 连接 Wi-Fi A（规则目标 SIM1）
2. 观察状态与通知是否切到 SIM1
3. 切换到 Wi-Fi B（规则目标 SIM2）
4. 观察是否切到 SIM2

## 6. 常见问题

### Q1: 检测不到 Wi-Fi

- 检查是否授予定位与附近设备权限
- 检查系统“定位总开关”是否开启

### Q2: 规则命中但不切卡

- 确认 LSPosed 作用域已勾选
- 某些 ROM 需要手动额外勾选 `android` 作用域
- 检查模块启用状态与系统兼容性

### Q3: 如何导出日志

- 在 App 内点击导出
- 或查看：`Android/data/com.laros.lsp.traffics/files/logs/`
